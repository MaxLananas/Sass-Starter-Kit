# Sass Starter Kit

A complete Sass project boilerplate demonstrating variables, mixins, loops, components, animations, utility classes, and responsive design. Built as a reference implementation and learning resource for Sass/SCSS architecture.

---

## Overview

This project provides a structured Sass codebase organized following a partial-based architecture. It compiles down to a single CSS file and includes a demo HTML page showcasing every component and utility class generated by the source files.

The codebase covers the major features of Sass: variables, nesting, mixins with parameters, loops (`@for`, `@each`), maps, conditionals (`@if`), partials, and `@import`-based file organization.

---

## Project Structure

```
project/
├── index.html              Demo page
├── css/
│   └── style.css           Compiled output (do not edit directly)
├── scss/
│   ├── style.scss           Main entry point (imports all partials)
│   ├── _variables.scss      Colors, spacing, typography, breakpoints, z-index
│   ├── _mixins.scss         Reusable functions (responsive, flex, buttons, cards, etc.)
│   ├── _reset.scss          Browser reset and base element styles
│   ├── _layout.scss         Container, grid system, sections, hero, footer
│   ├── _navigation.scss     Sticky navbar with BEM structure and mobile burger
│   ├── _components.scss     Buttons, cards, badges, alerts, avatars, forms, loaders, tooltips
│   ├── _animations.scss     Keyframes and animation utility classes
│   └── _utilities.scss      Generated utility classes (spacing, text, display, flex, etc.)
```

### Import Order

The main `style.scss` file imports partials in a specific order:

1. **Variables** and **Mixins** -- these are dependencies used by all other files
2. **Reset** -- normalizes browser defaults
3. **Layout** -- structural styles (container, grid, sections)
4. **Components** -- interface elements (buttons, cards, forms, etc.)
5. **Animations** -- keyframe definitions and animation classes
6. **Utilities** -- single-purpose classes, imported last so they can override other styles

## What Is Included

### Variables (`_variables.scss`)

All design tokens are centralized in one file:

- **Colors**: raw palette, semantic aliases (primary, success, danger, etc.), text colors, background colors
- **Spacing**: 8px grid scale (`4px` through `64px`)
- **Typography**: font families (sans-serif, serif, monospace), modular size scale (`0.75rem` to `3.5rem`), font weights, line heights
- **Borders and Radii**: small through pill-shaped radii, default border definition
- **Shadows**: five levels of elevation plus colored glow variants
- **Transitions**: preset easing curves including a bounce function
- **Breakpoints**: mobile (480px), tablet (768px), laptop (1024px), desktop (1200px), wide (1400px)
- **Dimensions**: max widths, navbar height, sidebar width
- **Z-index scale**: dropdown through tooltip layers

### Mixins (`_mixins.scss`)

Reusable style functions:

- `responsive($breakpoint)` -- wraps content in a `min-width` media query using named breakpoints
- `flex-center` -- shorthand for centered flexbox
- `flex(...)` -- configurable flexbox with direction, justify, align, wrap, and gap parameters
- `bouton($bg-color, $text-color, $taille)` -- generates a complete button style with hover, active, focus, and disabled states; supports three sizes
- `bouton-outline($color)` -- outline variant of the button mixin
- `card($padding)` -- card component with shadow and hover elevation
- `tronquer($lignes)` -- text truncation with ellipsis, supporting single and multi-line
- `background-image($url, $position, $size)` -- background image shorthand
- `aspect-ratio($width, $height)` -- enforces aspect ratio with `object-fit: cover` on child images
- `degrade($couleur1, $couleur2, $direction)` -- linear gradient shorthand
- `glassmorphism($opacity, $blur)` -- frosted glass effect using `backdrop-filter`
- `taille($width, $height)` -- width and height shorthand

### Reset (`_reset.scss`)

- `box-sizing: border-box` on all elements
- Margin and padding reset
- Smooth scroll behavior
- Base body typography
- Responsive images by default
- Link, list, heading, form element, and table resets

### Layout (`_layout.scss`)

- `.container` -- centered, max-width constrained content wrapper with narrow and fluid variants
- `.grid` -- CSS Grid system with 2, 3, and 4 column presets, an auto-fit responsive variant, and individual `.col-1` through `.col-12` span classes
- `.section` -- vertical padding with dark and gradient background variants, plus a centered header sub-component
- `.hero` -- full-viewport introductory section with gradient background, overlay, and content layout
- `.footer` -- dark footer with 4-column grid layout and responsive breakpoints

### Navigation (`_navigation.scss`)

- Sticky navbar using BEM naming (`.navbar`, `.navbar__container`, `.navbar__logo`, `.navbar__links`, `.navbar__link`, `.navbar__burger`)
- Active link indicator with pseudo-element underline
- Mobile burger menu toggle (display switching at tablet breakpoint)

### Components (`_components.scss`)

- **Buttons**: primary, secondary, success, danger, warning, dark, outline, small, large, full-width, icon (circular), pill, and button groups
- **Cards**: image with aspect ratio, tag, title, truncated text, footer with avatar; hover effect with image zoom
- **Badges**: generated via `@each` loop over a color map, with transparent background tint
- **Alerts**: left-bordered notifications generated via loop, with close button
- **Avatars**: four sizes (sm, default, lg, xl) with circular crop; overlapping avatar group with hover elevation
- **Forms**: labeled form groups, styled inputs with focus ring, error and success states, textarea support, help text
- **Loader/Spinner**: CSS-only spinner using border and `spin` animation, three sizes
- **Tooltips**: CSS-only tooltips using `::after` pseudo-element and `data-tooltip` attribute

### Animations (`_animations.scss`)

Keyframe definitions:
- `fadeIn`, `fadeInUp`, `fadeInLeft` -- opacity and position transitions
- `zoomIn` -- scale transition
- `bounce` -- vertical oscillation (infinite)
- `spin` -- 360-degree rotation (infinite, used by loader)
- `pulse` -- scale and opacity oscillation (infinite)
- `shake` -- horizontal vibration (for error feedback)

Utility classes: `.anim-fadeIn`, `.anim-fadeInUp`, `.anim-fadeInLeft`, `.anim-zoomIn`, `.anim-bounce`, `.anim-spin`, `.anim-pulse`, `.anim-shake`

Stagger delays: `.anim-delay-1` through `.anim-delay-10` (increments of 100ms)

### Utilities (`_utilities.scss`)

All generated via Sass loops and maps:

- **Spacing**: `.m-0` through `.m-6`, `.p-0` through `.p-6`, directional variants (`.mt-`, `.mr-`, `.mb-`, `.ml-`, `.mx-`, `.my-`, `.px-`, `.py-`), `.mx-auto`, `.ml-auto`, `.mr-auto`
- **Text**: alignment, weight, style, transform, size classes, color classes (`.text-primary`, `.text-danger`, etc.)
- **Background**: `.bg-primary`, `.bg-danger`, etc.
- **Display**: `.d-none`, `.d-block`, `.d-flex`, `.d-grid`, etc.
- **Flexbox**: `.flex-center`, `.flex-wrap`, `.flex-column`, `.justify-*`, `.align-*`
- **Gap**: `.gap-0` through `.gap-6`
- **Dimensions**: `.w-100`, `.w-75`, `.w-50`, `.w-25`, `.h-100`, `.w-screen`, `.h-screen`
- **Border radius**: `.rounded-sm` through `.rounded-pill`, `.rounded-none`
- **Borders**: `.border`, `.border-none`
- **Shadows**: `.shadow-none`, `.shadow-sm`, `.shadow`, `.shadow-lg`, `.shadow-xl`
- **Responsive visibility**: `.hidden-mobile`, `.visible-mobile`
- **Misc**: `.overflow-hidden`, `.cursor-pointer`, `.select-none`, `.opacity-50`, `.opacity-75`, `.divider`

## Usage

### Prerequisites

A Sass compiler is required. Options include:

- **Dart Sass** (recommended): `npm install -g sass`
- **Node-sass**: `npm install -g node-sass`
- Any IDE extension that compiles SCSS (VS Code Live Sass Compiler, etc.)

### Compilation

One-time build:

```bash
sass scss/style.scss css/style.css
```

Watch mode (recompiles on save):

```bash
sass --watch scss/style.scss:css/style.css
```

Production build with compression:

```bash
sass scss/style.scss css/style.css --style=compressed
```

### Viewing the Demo

Open `index.html` in a browser after compiling the CSS. The demo page includes examples of every component: navbar, hero section, badges, alerts, feature cards, buttons (all variants), article cards with avatars, avatar groups, a contact form, loaders, tooltips, and utility class demonstrations.

---

## Customization

To adapt this boilerplate to a different project:

1. Edit `_variables.scss` to change the color palette, font stack, spacing scale, or breakpoints. All components reference these variables, so changes propagate automatically.
2. Add or remove component partials as needed and update the import list in `style.scss`.
3. Extend mixins or add new ones in `_mixins.scss`.
4. The utility class generation in `_utilities.scss` can be extended by adding values to the `$espacements` or `$couleurs-texte` maps.
